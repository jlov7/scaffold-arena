# Security Policy

## Supported Versions

This repository is an active R&D project. Security fixes are applied to the latest `main` branch.

## Reporting a Vulnerability

Please do **not** open public GitHub issues for potential vulnerabilities.

Report security concerns privately with:
- A clear description of the issue
- Reproduction steps
- Impact assessment
- Suggested mitigation (if available)

Contact channel:
- Use your private coordination channel with the maintainer.

## Response Expectations

Target response times:
- Acknowledgement: within 3 business days
- Initial triage: within 7 business days
- Remediation timeline: communicated after triage

## Scope

In scope:
- Backend auth, input validation, and run execution controls
- Frontend token handling and injection surfaces
- Dependency vulnerabilities
- Infrastructure/configuration exposures

Out of scope:
- Social engineering requests
- Third-party provider outages

## Safe Harbor

Good-faith security research coordinated through the private channel above will be treated as authorized.
