# Tone and Voice

## Purpose
Define the content style for Scaffold Arena so every user-facing string is consistent, concise, and action-oriented.

## Principles
- Lead with outcome: explain what the user gets, not internal implementation details.
- Prefer direct verbs: `Run arena`, `Export report`, `Retry`.
- Keep error copy actionable: state what failed and what to do next.
- Avoid ambiguity: use explicit nouns (`run`, `comparison`, `report`) instead of vague phrasing.

## Writing Rules
- Labels and CTAs: 1-3 words where possible.
- Helper text: one sentence, concrete next action.
- Empty states: one educational line plus one primary CTA.
- Errors: include remediation in the same block.

## Applied Examples
- `Cannot start run while offline. Reconnect and retry.`
- `Connection failed.` + `Retry`
- `Start an arena run to capture your first benchmark and compare scaffold performance over time.`
